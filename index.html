Prework Study Guide

HTML

  • The <title> element defines the title of the webpage and is important for a search engine optimization (SEO, i.e., Google or Bing).

  • The <style> element defines the CSS styles associated with a website.

  • The <base> element defines the base URL for a webpage.

  • The <link> element connects an external resource to the HTML document.

  • The <meta> element defines metadata such as the character set, description, keywords, author, and viewport.

  • The <script> element embeds client-side scripts in the HTML document

CSS

  • When a developer wants to style an element directly in the HTML file, they use inline CSS. Inline styles apply to a specific HTML tag, using a style attribute with a CSS rule to style a particular page element only. An example of inline CSS is using the “style” attribute to change the color of a single “h1” heading:

    • <h1 style="color:white;">Prework Study Guide </h1>

  • It is also possible to embed an entire stylesheet directly in the HTML file using a <style> element in the <head> element. This internal style sheet contains CSS rules for the webpage in the head section of the HTML file.

  • An external CSS style sheet is preferred in most cases because it allows developers to keep all their CSS rules in a separate file, which makes design changes easier. When using an external style sheet, we link the file using a <link> element placed in the HTMLs <head>.

  • Separation of concerns is a widely used term in web development and refers to separating code into different files based on use.

  • Declarations contain two important components: the CSS property we want to apply and the value of the property. The declaration “color: blue;” includes the CSS property “color” and the value for that property, “blue.”

  • A <link> element creates the connection to the external file.

  • The <rel> attribute specifies the relationship between the current document and the linked document resource.

  • A class attribute adds styles to certain elements, but not all elements. They allow us to share a CSS rule to any element we choose by assigning the rule to a class attribute with a class selector.

  • A “margin” property acts much like a margin in a word processing document, it indicates how much space we want around the outside of an element. When it is set to 0, we are indicating that we want NO extra space around the element.

  • The “padding” property is often confused with “margin,” but it is not the same thing. It adds space around the content inside an element. When you increase padding, you are making the entire element bigger.

Git

  • git status: checks what branch we are currently on

  • git checkout -b branch-name: creates a new branch and switches to it

JavaScript

  • JavaScript is a high-level dynamic programming language that allows users to interact with website. With JS you can power your websites with effects, interactivity, data communication, and much more.

  • A variable is a named container that allows us to store data in our code. To declare a variable, we use the “var” keyword. After the “var” keyword, we give the variable a name. This name must be unique because it is how we will reference the information stored in the variable in our code. Finally, we use an assignment operator “=” to assign the specific data that we want stored in our variable.

    • var topics = "HTML, CSS, Git, JavaScript";

  • The “topics” variable contains a string that includes all the topics that we are studying in this module. All strings are surrounded by quotation marks, and each line of JavaScript ends with a semicolon. To access the string “HTML, CSS, Git, JavaScript” we can simply refer to the variable’s name, “topics.”

  • Use the “console.log” method to output a message to the web console by adding an argument. In this case, “console.log” will output the value of “topics” to the web console because we’ve added it as an argument inside the parenthesis.

  • Control flow is the order in which a computer executes code in a script. In JavaScript, code runs from the first statement at the top of the page, down to the last statement at the bottom of the page. We can use conditional statements and loops to control the order in which the code is executed, or if it is even executed at all.

  • The conditional statement, “if,” means the code will only execute it the statement meets a condition or is true. If one condition is truthy, the computer will execute the code that we provide between the curly brackets. If the condition is not truthy, the code will not execute, interrupting the control flow.

  • An array is a single variable that is used to hold a group of data. Arrays are typically used to hold data that is related in some way. Arrays are especially useful when we want to perform a task on multiple pieces of data at once.

  • A “for” loop uses the predictable pattern of indices to perform a task on all the items in an array by allowing a single code block to be executed over and over.

  • Commenting out code makes it inactive without deleting it, to do this in JS, put “/*” before and “*/” after the block of code.

  • A function is a set of instructions that tells the computer how to perform a certain task. A function’s name is followed by parentheses, “().”
